# 静态路由

## 1.1 什么是静态路由？

静态路由是指路径固定的路由，通过创建固定的文件夹结构来定义。

在 Next.js App Router 中，**文件夹结构 = URL 路径**。

---

## 1.2 创建静态路由

### 基本步骤

1. 在 `app/` 目录下创建文件夹
2. 在文件夹内创建 `page.tsx` 文件
3. 文件夹名称就是 URL 路径

### 示例

**创建关于页面：**

```text
app/
└── about/
    └── page.tsx
```

**对应的 URL：** `/about`

**创建联系页面：**

```text
app/
└── contact/
    └── page.tsx
```

**对应的 URL：** `/contact`

---

## 1.3 实际示例

### 示例 1：关于页面

**第一步：创建文件夹和文件**

```text
app/
└── about/
    └── page.tsx
```

**第二步：编写页面组件**

```typescript
// app/about/page.tsx
export default function AboutPage() {
  return (
    <div>
      <h1>关于我们</h1>
      <p>这是关于页面的内容</p>
    </div>
  )
}
```

**访问 URL：** `http://localhost:3000/about`

---

### 示例 2：联系页面

**第一步：创建文件夹和文件**

```text
app/
└── contact/
    └── page.tsx
```

**第二步：编写页面组件**

```typescript
// app/contact/page.tsx
export default function ContactPage() {
  return (
    <div>
      <h1>联系我们</h1>
      <form>
        <input type="text" placeholder="姓名" />
        <input type="email" placeholder="邮箱" />
        <textarea placeholder="留言"></textarea>
        <button type="submit">提交</button>
      </form>
    </div>
  )
}
```

**访问 URL：** `http://localhost:3000/contact`

### 示例 3：嵌套静态路由

**创建产品页面：**

```text
app/
└── products/
    ├── page.tsx          # /products
    └── featured/
        └── page.tsx      # /products/featured
```

**产品列表页：**

```typescript
// app/products/page.tsx
export default function ProductsPage() {
  return (
    <div>
      <h1>产品列表</h1>
      <p>所有产品</p>
    </div>
  )
}
```

**特色产品页：**

```typescript
// app/products/featured/page.tsx
export default function FeaturedProductsPage() {
  return (
    <div>
      <h1>特色产品</h1>
      <p>精选产品推荐</p>
    </div>
  )
}
```

**访问 URL：**
- `http://localhost:3000/products` - 产品列表
- `http://localhost:3000/products/featured` - 特色产品

## 1.4 关键规则

### ✅ 必须遵循的规则

1. **必须有 `page.tsx`**：没有 `page.tsx` 的文件夹不会创建路由
2. **文件夹名称 = URL 路径**：`app/about/page.tsx` → `/about`
3. **大小写敏感**：`About` 和 `about` 是不同的路由

### ❌ 常见错误

**错误 1：忘记创建 `page.tsx`**

```text
app/
└── about/          # ❌ 没有 page.tsx，不会创建路由
    └── info.tsx
```

**正确做法：**

```text
app/
└── about/
    └── page.tsx    # ✅ 必须有 page.tsx
```

**错误 2：使用错误的文件名**

```text
app/
└── about/
    └── AboutPage.tsx  # ❌ 文件名必须是 page.tsx
```

**正确做法：**

```text
app/
└── about/
    └── page.tsx    # ✅ 文件名必须是 page.tsx
```

## 1.5 静态路由 vs 动态路由

| 特性 | 静态路由 | 动态路由 |
|------|---------|---------|
| **语法** | 普通文件夹名 | `[param]` |
| **路径** | 固定 | 根据参数变化 |
| **示例** | `/about` | `/blog/[id]` |
| **使用场景** | 固定数量的页面 | 内容数量不固定 |

## 1.7 总结

### 关键要点

1. **静态路由**：通过固定的文件夹结构创建
2. **必须包含 `page.tsx`**：没有 `page.tsx` 不会创建路由
3. **文件夹名称 = URL 路径**：`app/about/page.tsx` → `/about`
4. **适用于固定页面**：关于、联系、服务等固定数量的页面
