> 📖 **详细讲解**：请查看 [02-服务端组件实践.md](./02-服务端组件实践.md)

## 3. 客户端组件实践

> 📖 **详细讲解**：请查看 [03-客户端组件实践.md](./03-客户端组件实践.md)

### 快速要点

## 总结

### 关键概念回顾

1. **组件目录结构**：合理的目录组织提高代码可维护性
2. **服务端组件**：默认组件类型，在服务器上运行，可直接获取数据
3. **客户端组件**：使用 `'use client'`，支持交互性和 Hooks
4. **组件组合**：通过 props 和 children 实现组件复用
5. **样式管理**：使用 Tailwind CSS 进行样式开发

### 最佳实践

- ✅ 默认使用服务端组件，只在需要交互时使用客户端组件
- ✅ 合理组织组件目录结构，区分共享组件和局部组件
- ✅ 使用 TypeScript 定义组件 props 类型
- ✅ 创建可复用的基础组件，提高开发效率
- ✅ 使用 Tailwind CSS 进行样式开发，保持样式一致性

完成第三阶段后，可以进入第四阶段：**数据获取**，学习如何在服务端组件中获取数据、处理加载状态和错误。
